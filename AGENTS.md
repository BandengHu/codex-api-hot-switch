# 项目约束

## 原样透传链路红线

当供应商启用 `rawResponsesPassthrough` 时，这条链路只能做两件事：

- 按当前热切配置替换请求里的模型。
- 按当前热切配置替换或删除请求里的 reasoning / 推理强度字段。

除此之外，原样透传链路不能做任何协议、内容或流式事件改动，包括但不限于：

- 不能注入语言策略、系统提示、行动说明或任何额外 message。
- 不能做 chat / Claude / Responses 协议转换。
- 不能补齐、改写或重排 `response.output`、`output_index`、`phase`、tool call、reasoning、usage。
- 不能修改上游 SSE 事件顺序、事件类型、终止帧或正文内容。
- 不能把非透传链路的修复逻辑复用到透传链路。

如果某个问题只在非透传链路出现，只能在对应非透传转换模块里修复。不要为了修其它供应商或模型，把修复逻辑接到原样透传链路上。

如确实需要改变原样透传链路边界，必须先明确说明风险并征得用户确认。
